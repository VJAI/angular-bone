angular-bone

Introduction

angular-bone helps to build page layouts using CSS3 Grid and Flex concepts for your angular apps. It provides you a set of directives that helps to build responsive layouts in a declarative way.

As default, it follows bootstrap's breakpoint system that allows you to configure layouts for 5 different screen sizes (Extra Small, Small - sm, Medium - md, Large - lg and Extra Large - xl). It also follows mobile-first approach.

The library also provides you a media-size watcher service that you can use anywhere in your application to listen to the breakpoint changes and act accordingly.

Installation

You can install the library from npm. 

npm install angular-bone --save

Flex Layouts

You can create flex layouts using two directives: bon-f and bon-fi.  bon-f is used to configure the properties of a flex container and bon-fi is used to configure the properties of a flex item.

Simple flex layout

Code

<div bon-f>
  <div class="green" bon-fi bon-fi-grow="1"></div>
  <div class="yellow" bon-fi bon-fi-grow="1"></div>
  <div class="pink" bon-fi bon-fi-grow="2"></div>
</div>

Preview

Responsive flex layout

You can configure the properties for a particular screen size by suffixing the media size abbreviation (sm, md, lg and xl) after the property name. If you don't specify any suffix then it's assumed for the extra small screen. If you want the same configuration for all the screen sizes then ignore the suffix.

Code

<div bon-f
     bon-f-direction="column"
     bon-f-direction-md="row">
  <div class="green" bon-fi bon-fi-grow="1"></div>
  <div class="yellow" bon-fi bon-fi-grow="1"></div>
  <div class="pink" bon-fi bon-fi-grow="1" bon-fi-grow-md="2"></div>
</div>

Grid Layouts

You can create awesome grid layouts using two directives: bon-g and bon-gi. bon-g is used to configure the properties of a grid container and bon-gi is used to configure the properties of a grid item.

Simple grid layout

Code

<div bon-g
     bon-g-cols="30% 1fr"
     bon-g-rows="20% 1fr 10%"
     bon-g-area="'a a' 'b c' 'd d'">
  <div class="purple" bon-gi bon-gi-area="a"></div>
  <div class="yellow" bon-gi bon-gi-area="b"></div>
  <div class="green" bon-gi bon-gi-area="c"></div>
  <div class="pink" bon-gi bon-gi-area="d"></div>
</div>

Preview

Responsive grid layout

Code

<div bon-g
     bon-g-cols="1"
     bon-g-cols-md="30% 1fr"
     bon-g-rows="20% 30% 1fr 10%"
     bon-g-rows-md="20% 1fr 10%"
     bon-g-area="'a' 'b' 'c' 'd'"
     bon-g-area-md="'a a' 'b c' 'd d'">
  <div class="purple" bon-gi bon-gi-area="a"></div>
  <div class="yellow" bon-gi bon-gi-area="b"></div>
  <div class="green" bon-gi bon-gi-area="c"></div>
  <div class="pink" bon-gi bon-gi-area="d"></div>
</div>

Mixing grid and flex

Code

<div bon-g
     bon-g-cols="1"
     bon-g-cols-md="30% 1fr"
     bon-g-rows="20% 30% 1fr 10%"
     bon-g-rows-md="20% 1fr 10%"
     bon-g-area="'a' 'b' 'c' 'd'"
     bon-g-area-md="'a a' 'b c' 'd d'">

  <div class="purple"
       bon-gi
       bon-gi-area="a"
       bon-f
       bon-f-direction="row"
       bon-f-align-items="center">
      <div class="grey"></div>
      <div class="grey"></div>
      <div class="empty" bon-fi bon-fi-grow="1"></div>
      <div class="grey"></div>
  </div>

  <div class="yellow"
       bon-gi
       bon-gi-area="b"
       bon-f
       bon-f-direction="column">
    <div class="grey" bon-fi bon-fi-grow="1"></div>
    <div class="grey" bon-fi bon-fi-grow="1"></div>
    <div class="grey" bon-fi bon-fi-grow="1"></div>
  </div>

  <div class="green"
      bon-gi
       bon-gi-area="c"
       bon-f
       bon-f-direction-md="column">
    <div class="grey" bon-fi bon-fi-grow="1"></div>
    <div class="grey" bon-fi bon-fi-grow="2"></div>
    <div class="grey" bon-fi bon-fi-grow="1"></div>
  </div>

  <div class="pink" bon-gi bon-gi-area="d"></div>
</div>

API

Flex Directives

You can use bon-f and bon-fi directives to do flex layout system for your pages.

Below tables lists the properties of both the directives. If default is not provided then it's assumed to be the browser's default.

Properties of flex container directive bon-f
Property Name Type Default Description
bon-f string flex The display type of the container. The value can be either "flex", "inline-flex" or "none".
bon-f-direction string The flex direction. It can be one of these values: "row", "row-reverse", "column" or "column-reverse".
bon-f-wrap string Controls the wrapping behavior of flex items. It can be "nowrap", "wrap" or "wrap-reverse".
bon-f-justify-content string Defines the alignment along the main axis. Some of the possible values are: "flex-start", "flex-end", "center" etc.
bon-f-align-items string Defines the default behaviour for how flex items are laid out along the cross axis. Some of the possible values are: "flex-start", "flex-end", "center" etc.
bon-f-align-content string Defines the alignment of individual items within the main-axis. Some of the possible values are: "flex-start", "flex-end", "center" etc.

The above properties repeats for other screen sizes with the respective suffix. For example, in case of small screen the properties are: bon-f-sm, bon-f-direction-sm and so on.

Properties of flex item directive bon-fi
Property Name Type Default Description
bon-fi-order number Controls the order in which they appear in the flex container.
bon-fi-grow Defines the ability for a flex item to grow.
bon-fi-shrink Defines the ability for a flex item to shrink.
bon-fi-basis Defines the default size of an element before the remaining space is distributed.
bon-fi-align The default alignment of the item.

The above properties repeats for other screen sizes with the respective suffix.

Properties of grid directive bon-g
Property Name Type Default Description
bon-g string The display type of grid container. The value can be either "grid", "inline-grid" or "none".
bon-g-cols number | Array<number | string> | string The columns configuration of grid. Passing number create that many equal size columns. You can also pass an array of column values or a string.
bon-g-rows number | Array<number | string> | string The columns configuration of grid. Passing number create that many equal size rows. You can also pass an array of column values or a string.
bon-g-area Array<Array<string>> | string The area configuration. You can pass as an array of arrays or just a string.
bon-g-auto-cols number | string The default size of auto-generated grid columns.
bon-g-auto-rows number | string The default size of auto-generated grid rows.
bon-g-auto-flow string Controls how auto-placed items get inserted in the grid. It's an enumeration and the values are "row", "column", "row dense" and "column dense".
bon-g-gap number | Array<number | string> | string The gap between rows and columns.
bon-g-justify-items string Specifies the alignment of grid items along the row axis. Some of the possible values are "start", "end", "center" and "stretch".
bon-g-align-items string Specifies the alignment of grid items along the column axis. Some of the possible values are "start", "end", "center" and "stretch".
bon-g-justify-content string Specifies the alignment of grid along the row axis.
bon-g-align-content string Specifies the alignment of grid along the column axis.
Properties of grid directive bon-g
Property Name Type Default Description
bon-gi-col number | [number | string, number | string] | string Passing a number will specify the spanning of the column. You can also specify the position as an array or string.
bon-gi-row number | [number | string, number | string] | string Passing a number will specify the spanning of the row. You can also specify the position as an array or string.
bon-gi-area [number | string, number | string, number | string, number | string] | string The area of the item.
bon-gi-justify string Specifies the alignment of a grid item inside a cell along the inline (row).
bon-gi-align string Specifies the alignment of a grid item inside a cell along the inline (column).
